1. ¿Qué es MQTT y por qué nos importa?

Entendiendo el protocolo del Internet de las Cosas (IoT).

Esta sección introduce el concepto fundamental de MQTT, diseñado para situaciones donde el ancho de banda es bajo o donde los dispositivos tienen pocos recursos, superando las limitaciones del HTTP tradicional en entornos industriales.

La diferencia clave: Pub/Sub vs Polling

En la web normal (HTTP), tu navegador pregunta constantemente: "¿Hay datos nuevos?". Esto satura la red. MQTT usa Publicar/Suscribir (Pub/Sub), similar a las notificaciones de tu celular: los datos llegan solos cuando se publican.

Comparativa Teórica de Tráfico de Red (Ejemplo de 1 hora)

HTTP realiza peticiones constantes. MQTT mantiene una conexión liviana y solo transmite cuando hay un evento.

3. Mosquitto: Nuestro Broker

El núcleo de la red local.

En este proyecto utilizaremos Eclipse Mosquitto, un broker de código abierto que implementa el protocolo MQTT de manera eficiente.

  • ✓ Súper liviano: Puede correr en una PC de laboratorio o en una Raspberry Pi.
  • ✓ Es el estándar: Fundamental para cualquier sistema IoT industrial.
  • ✓ Herramientas CLI: Incluye utilidades de consola para pruebas rápidas.

// Herramientas incluidas para probar antes de programar

# Para publicar un mensaje:
mosquitto_pub -t "test/tema" -m "Hola"
# Para escuchar mensajes:
mosquitto_sub -t "test/tema"

4. Entendiendo los Tópicos

El sistema de enrutamiento de mensajes.

El Broker organiza la información en Tópicos. Son como rutas de carpetas que identifican de qué trata el mensaje.

Comodines (Wildcards)
